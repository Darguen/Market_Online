Requerimientos funcionales:

El usuario debe poder iniciar sesión
El usuario debe gestionar el inventario disponible
El usuario debe ver el carrito de compras antes de finalizar la compra
El usuario debe ver los productos disponibles que estan a la venta
El usuario debe poder agregar elementos al carrito
El usuario debe poder eliminar elementos del carrito


Requerimientos no funcionales:

La interfaz debe ser intuitiva para el usuario
La aplicacion debe incorporar imágenes
La aplicacion debe mostrar mensajes para verificar que se borraron y agregaron elementos exitosamente

Descripcion del proyecto:

Se requiere realizar una aplicación movil que permita a los usuarios realizar el registro de mercaderia(comida, articulos de aseo, etc), para mantener un registro
completo de lo que ya se tiene comprado y lo que falta por comprar.

Por esto se pide que la aplicación le permita al usuario registrarse a la aplicación, gestionar el inventario de productos, manejar la agrupacion de la mercaderia, ver los precios asociados y el presupuesto disponible.
Todo lo antes mencionado se necesita mediante el uso de interfaces interactivas en pantallas tales como, inventario, carrito de compras, lugares frecuentes, etc.
